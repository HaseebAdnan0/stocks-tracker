## Iteration 6 - US-006: Create session management utilities

### Status: ✓ COMPLETED

### Acceptance Criteria:
- [x] Add `createSession(userId)` returning session token
- [x] Add `validateSession(token)` returning user or null
- [x] Add `deleteSession(token)` for logout
- [x] Sessions expire after 7 days
- [x] Typecheck passes

### Changes Made:
1. Updated psx-tracker/lib/auth.ts with three session management functions:
   - createSession(userId: number): Promise<string>
     * Generates secure random token using crypto.randomBytes(32)
     * Calculates expiration date 7 days in the future
     * Inserts session record into database with user_id, token, expires_at
     * Returns the generated token

   - validateSession(token: string): Promise<User | null>
     * Queries sessions table joined with users table
     * Checks token matches AND expires_at > current datetime
     * Returns user object if valid, null if expired or not found

   - deleteSession(token: string): Promise<void>
     * Deletes session record from database by token
     * Used for logout functionality

2. Added imports to auth.ts:
   - crypto module for secure token generation
   - getDb and User type from './db' for database operations

3. Verified: TypeScript compilation successful (npx tsc --noEmit)

### Files Modified:
- psx-tracker/lib/auth.ts

### Learnings:
- crypto.randomBytes(32).toString('hex') generates a 64-character hex token (32 bytes = 256 bits)
- SQLite datetime('now') function for comparing expiration timestamps in queries
- JOIN query pattern efficiently validates session and retrieves user in single query
- Session expiration checked at validation time, expired sessions remain in DB but are ignored
- All functions remain async for consistency with existing auth.ts functions
