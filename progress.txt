### US-033: Implement dividend history scraper
- fetch_dividend_history(symbol) function follows same dual-format pattern (JSON first, then HTML fallback)
- Primary source is https://dps.psx.com.pk/company/{SYMBOL} endpoint (same as fundamentals)
- Dividend fields include: announcement_date, type (Cash/Bonus/Right), amount
- Return empty list [] if no dividends found (different from other scrapers that return None)
- Return None only if request fails (network error, timeout, etc.)
- JSON parsing checks multiple possible keys: "data", "dividends", "dividend_history", "corporate_actions"
- Filter out non-dividend corporate actions by checking for keywords: cash, bonus, right, dividend, div
- Skip entries where amount is 0 or negative (invalid dividends)
- Sort by date descending (reverse=True) to show most recent dividends first
- HTML parsing looks for tables with dividend-related keywords in headers
- Strategy 1: Find table by checking header text for keywords (dividend, announcement, corporate action)
- Strategy 2: Find section headers (h2, h3, h4, div) with dividend keywords, then find next table
- HTML row parsing extracts: date (first column), type (cell with Cash/Bonus/Right keywords), amount (numeric value)
- _extract_numeric_from_text() helper used to extract amount from text containing numbers and symbols
- Default dividend type is "Cash" if not explicitly found in HTML
- Both JSON and HTML parsers handle missing/malformed data gracefully with try/except around each row
- logging.info() used for successful parsing, logging.warning() for row-level errors, logging.error() for fatal errors
- Type hints: Optional[List[Dict[str, Any]]] for return value (None or list of dicts)
- Function returns [] (empty list) when no dividend data found, maintaining consistency with "no data" case
